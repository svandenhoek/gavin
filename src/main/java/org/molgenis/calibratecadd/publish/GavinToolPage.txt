
<h1>Please upload a VCF file (GRCh37/hg19) containing up to 100,000 variants</h1>

<p>
Please provide a <a href="http://vcftools.sourceforge.net/specs.html">VCF file</a> of your variants. This web service follows the input rules of the <a href="http://cadd.gs.washington.edu/score">CADD service</a>, meaning that it is sufficient to provide the first 5 columns of a (gzipped) VCF file. All other information than CHROM, POS, REF, ALT will be ignored, as is the header (or any line starting with #). Files may be up to 3MB, and up to 100,000 can be processed per submission. Use file extension .vcf, .vcf.gz, .tsv or .tsv.gz.
</p>

<p>
Alternatively, you may provide a <a href="http://cadd.gs.washington.edu/score">CADD service</a> output file. This allows you to pre-score indels (and/or SNVs) for GAVIN classification. This format is detected as [CHROM POS REF ALT RawScore PHRED], the last two being decimals, headers are ignored. You may use any CADD-output file directly in GAVIN.
</p>

<p>
Your results will be available under an random URL that others cannot reach without knowing. Results are deleted after 24 hours and will not be accessed by the MOLGENIS team.
</p>

<p>
Example input files:<br>

<a href="http://www.molgenis.org/downloads/gavin/variantclassificationwebtool/testfiles/gavin_simpleexamplefile_fromcadd.tsv">CADD-output file with 6 columns and original header</a>
</p>
